## Цель работы
Реализовать алгоритм умножения матриц с использованием технологии OpenMP, провести серию экспериментов с различным количеством вычислительных ядер, проанализировать ускорение и эффективность параллелизации.

## Технологии
- **C++** (стандарт C++11/14/17) — основная реализация алгоритма
- **Python 3** — генерация тестовых матриц и визуализация результатов
- **OpenMP** - Директивы параллелизма #pragma omp parallel for
```
parallel_programming-2/
├── ConsoleApplication1.cpp      # Параллельная реализация (OpenMP)
├── run_exp.py                   # Автоматизация экспериментов
├── visualize_stats.py           # Построение графиков
├── matrix_generation.py         # Генератор тестовых матриц
├── config.txt                   # Конфигурация тестов (опционально)
│
├── # Входные/выходные данные (генерируются):
├── matrix_A.txt                 # Матрица A
├── matrix_B.txt                 # Матрица B
├── result_matrix.txt            # Результат умножения
├── results_stat.txt             # Статистика всех запусков
└── report.png                   # Итоговые графики
│
└── README.md                    # Этот файл
```

### Наблюдения:

✅ Максимальное ускорение: 5.77× (1600, 12 ядер)

✅ Среднее ускорение на больших матрицах: 4.5-5.7×

⚠️ Ускорение не линейное (при 12 ядрах идеал = 12×, реально = 5.77×)

### Причины нелинейности:

1.Накладные расходы на создание и синхронизацию потоков

2.Конкуренция за подсистему памяти и кэш

3.Дисбаланс нагрузки между потоками

4.Ограничения архитектуры (память, шина)
