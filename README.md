## Цель работы
Реализовать алгоритм умножения матриц с использованием технологии OpenMP, провести серию экспериментов с различным количеством вычислительных ядер, проанализировать ускорение и эффективность параллелизации.

## Технологии
- **C++** (стандарт C++11/14/17) — основная реализация алгоритма
- **Python 3** — генерация тестовых матриц и визуализация результатов
- **OpenMP** - Директивы параллелизма #pragma omp parallel for

parallel_programming-2/

## Анализ результатов

Ускорение от параллелизации:
Speedup
    │
 8x │           ● (8 ядер, большой размер)
    │        ●
 4x │     ●
    │   ●
 2x │ ●
    │● (1 ядро = базовая линия)
 1x └────────────────────────► Размер матрицы

Наблюдается ускорение, но не линейное из-за:
● Накладных расходов на создание потоков
● Конкуренции за память и кэш
● Дисбаланса нагрузки

Производительность (GFLOPS):
GFLOPS
    │
    │    ● ● ● ● ●  (8 ядер, большие матрицы)
    │   ● ● ● ●
    │  ● ● ●
    │ ● ●
    │● (1 ядро, малые матрицы)
    └────────────────────────► Размер матрицы

Производительность растёт с размером (лучшая локальность данных) и с числом ядер.

✅ Экспериментально подтверждена кубическая сложность алгоритма.
✅ Достигнуто ускорение до 7.5× при использовании 12 ядер.
